# 复盘摘要与人工决策支撑 V1 技术说明书

## 1. 文档目的
这份说明书记载 `codex/risk-history-v1` 分支在 `L. 复盘摘要与人工决策支撑 V1` 阶段新增的复盘摘要能力。

本阶段不做新的自动化执行，不做自动调策略，也不做自动转正/回滚；只解决一个问题：

> 现有复盘数据已经很多，但你需要更短、更直接的摘要和人工动作建议，能在命令行或 Telegram 管理入口里快速看到“现在该看什么、先做什么、先别做什么”。

---

## 2. 阶段定位
到 `K. Telegram 值守查询补强 V1` 为止，系统已经具备：

- 结构化历史资产与 `fp 1~6`
- 策略版本化、灰度、回滚
- 受控自学习的候选、影子、灰度、转正、回滚链路
- TG 值守播报与定向查询

当前缺的不是更多数据，而是更压缩的人工决策支撑：

- 24h 复盘要点能不能更快看完
- 结构化链路缺口能不能更快暴露
- 当前应该优先复查哪里，能不能给出人工建议

---

## 3. 设计边界

### 3.1 本阶段完成
1. `fp brief`
2. `fp gaps`
3. `fp action`
4. 保持输出短文本，适合管理员聊天入口和手机查看
5. 对应测试、文档、回归

### 3.2 本阶段明确不做
1. 自动 `policy sync`
2. 自动 `learn promote`
3. 自动 `learn rollback`
4. 自动调任务、调资金、调档位
5. 新增复杂报表平台

---

## 4. 命令设计

### 4.1 `fp brief`
输出压缩后的 24h 复盘摘要，重点只保留：

- 当前盘面与温度
- 当前策略版本与模式
- 当前学习阶段摘要
- 24h 样本、胜率、盈亏、回撤
- 24h 观望、阻断、相似样本、证据强度
- 链路覆盖摘要
- 当前运行态建议

目标是让你在几行内知道“当前大势是否健康、证据是否够、链路是否完整”。

### 4.2 `fp gaps`
输出压缩后的链路缺口摘要，重点关注：

- 盘面覆盖
- 决策覆盖
- 执行覆盖
- 结算覆盖
- 有盘面无决策
- 有决策无执行
- 有下注无结算
- 有结算无下注
- 有阻断无风控记录
- 有观望无策略记录

目标不是替代 `fp 6`，而是先快速回答：

> 这批样本现在能不能直接拿来判断策略/学习效果，还是要先修链路。

### 4.3 `fp action`
输出人工动作建议，只做“建议”，不做自动执行。

初版启发式包括：

1. 决策/执行/结算链路存在明显缺口时，先修链路再动策略
2. 高档位最近承压时，提醒优先复查档位表现
3. 当前相似样本更偏向低档/中档/观望且 24h 承压时，提醒先按保守口径复核
4. 灰度候选仍在运行且当前环境承压时，提醒人工复核是否需要回滚
5. 策略主动观望明显偏高时，提醒结合 `replay / explain` 复查 prompt 与策略摘要

目标是把已有复盘信息压缩成“下一步先看哪里”的入口，而不是替你做决策。

## 5. 实现方式

### 5.1 数据来源
本阶段不新增数据库表，直接复用已有数据：

- `build_current_analysis_snapshot(...)`
- `build_policy_evidence_package(...)`
- `_recent_analytics_context(...)`
- 当前运行态 `runtime`
- 学习中心当前候选状态

### 5.2 新增构建函数
在 `history_analysis.py` 新增：

- `build_fp_brief(user_ctx)`
- `build_fp_gap_brief(user_ctx)`
- `build_fp_action_report(user_ctx)`

同时补充内部聚合辅助函数：

- `_linkage_gap_stats(...)`
- `_tier_group_review(...)`
- `_review_learning_status(...)`

### 5.3 命令入口
在 `zq_multiuser.py` 的 `fp` 命令入口增加：

- `fp brief`
- `fp gaps`
- `fp action`

现有 `fp 1~6` 保持不变。

## 7. 测试范围
本阶段新增测试覆盖：

1. `fp brief` 摘要字段是否完整
2. `fp gaps` 是否正确暴露主要链路缺口
3. `fp action` 是否能在承压样本下输出人工建议
4. `fp brief / fp gaps / fp action` 命令是否正确路由

---

## 8. 本阶段结论
`L. 复盘摘要与人工决策支撑 V1` 完成后，复盘入口从“有很多明细”进一步收束到“能快速给出人工判断起点”：

- 可以先看短摘要，再决定要不要钻进 `fp 5 / fp 6`
- 可以先看链路缺口，再决定是否相信这批样本
- 可以直接收到人工动作建议，但不会越界到自动变更策略或学习状态

这也意味着当前 A 到 L 的既定阶段已经闭环，可以视为这条分支的初步 `v0.1.1` 基线。
