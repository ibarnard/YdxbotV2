# 任务模板与快速创建 V1 技术说明书

## 1. 文档目的
本说明书记录 `codex/risk-history-v1` 分支在“任务系统 V1”之后继续完成的模板化能力，重点说明：
- 任务模板定义
- 快速创建命令
- 代码入口
- 当前边界

---

## 2. 阶段定位
上一阶段的任务系统 V1 已经具备：
- `tasks.json`
- 任务状态机
- 盘面/定时/混合触发
- 任务日志与统计

但创建方式仍然偏工程参数化：
- `task add <名称> <预设> <局数> ...`

这一阶段的目标不是重做任务系统，而是把“常见任务的创建”简化成模板入口，降低日常使用成本。

---

## 3. 当前新增能力

### 3.1 新增命令
- `task tpl`
  - 查看内置任务模板
- `task new <模板> [名称]`

- 基准预设：`yc20`

- 亏损上限：`12000`

### 3.3 模板别名

## 5. 设计原则

### 5.1 模板负责默认值
模板只提供一组默认配置：
- 触发方式
- 盘面标签
- 基准预设
- 目标笔数
- 亏损上限

### 5.2 模板不绕开原有风控
模板创建出的任务，仍然受以下机制约束：
- `fk1`
- `fk2`
- `fk3`
- 动态档位
- 资金风控

### 5.3 默认创建后不自动开启
当前 `task new` 只负责创建，不默认直接运行。

后续由用户显式执行：
- `task on <id>`
- `task run <id>`

这样更安全，避免误建任务后立即接管真实下注。

---

## 6. 命令示例

查看模板：
```text
task tpl
```

按模板创建：
```text
task new 保守巡航
task new 趋势 巡航A
```

创建后继续：
```text
task list
task on <id>
task run <id>
```

---

## 7. 当前边界
本阶段没有做：
- 模板参数二次覆盖
- 向导式交互
- 模板编辑/保存自定义模板
- 模板与任务包联动

原因：
- 当前优先目标是把创建入口从“长参数命令”降成“模板 + 快速创建”
- 先做稳，再做更复杂的任务模板体系

## 9. 下一阶段建议
任务模板与快速创建之后，下一步更值得继续的是：

1. 任务模板支持可选覆盖参数
- 例如模板创建时覆盖预设、目标笔数、止损

2. 任务包
- 多个任务组成一个执行包，由盘面和收益状态决定切换

3. 更高层调度
- 任务优先级仲裁
- 任务冲突处理
- 多账号分层调度

当前最自然的下一步是：
- 任务包 V1

因为现在已经有：
- 风控底座
- 动态押注
- 单任务系统
- 模板化创建

下一步就可以开始把多个任务组织成一个更接近实战的任务包。
